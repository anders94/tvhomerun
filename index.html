About the Project

TVHomeRun is a family of apps for accessing and streaming recorded content from your HDHomeRun DVR devices. It consists of a backend server that automatically discovers your DVR devices and converts recordings to streaming-ready formats, plus native applications for all your devices.

With TVHomeRun, you can browse your recorded shows, resume playback where you left off, and enjoy your content on Apple TV, iPhone, iPad, or any web browser.

Background

I've found few interfaces as inscrutable as the native HDHomeRun applications, both for iPhone and AppleTV. Beyond my troubles finding and doing what I want, the video delay when you skip forward and backward in time to get around commercials (probably one of the most common things you do) leaves much to be desired. After skipping, you'll get the audio fairly quickly but the video takes a second or two to catch up. Not the end of the world but it makes me pine for the days of the legendary TiVo responsivness where you can instantly blast both forward and backward through your recordings with impunity. I've seen Apple products do this though, just not with the native HDHomeRun or related apps.

It turns out mobile devices (Apple's in particular) use HLS for streaming which is highly optimized for scrubbing quickly. Hence, I vibe coded TVHomeRun which includes a server to convert HDHomeRun DVR shows into HLS streams and a suite of applications to consume them. Currently, you'll find iPhone, AppleTV and web versions.

Because HDHomeRun devices don't directly create HLS streams, a proxy server is used to transcode recorded episodes from an HDHomeRun DVR and serve them to clients as HLS streams. Several API endpoints provide show and episode listings as well as tools to update watch progress. The HDHomeRun device is always used as the source of truth for recordings and storing watch progress.

The result is a set of Apple compatible apps that strive for TiVo-like video responsivness. At the moment, the applications are only for viewing programs saved in the DVR but live TV interaction is possible in a future version.

Quick Start Guide

Step 1: Set Up the Backend Server

# Clone the backend repository
git clone https://github.com/anders94/tvhomerun-backend.git
cd tvhomerun-backend

# Install dependencies
npm install

# Start the server (default port 3000)
npm start

Requirements: Node.js v14+ and at least one HDHomeRun device with DVR on your local network.

The server will automatically discover your HDHomeRun devices and sync content every hour. Access the API at http://your-server-ip:3000

Step 2: Choose Your Client Application

üì∫ Apple TV (tvOS)
  1. Clone tvhomerun-tvos repository
  2. Open in Xcode 14.0+
  3. Build and run on your Apple TV
  4. Configure server URL on first launch
üì± iPhone/iPad (iOS)
  1. Clone tvhomerun-ios repository
  2. Open in Xcode 14.0+
  3. Build and install on your device
  4. Enter server URL in settings
üåê Web Browser
  1. Clone tvhomerun-web repository
  2. Run npm install
  3. Start with npm start
  4. Access at http://localhost:8080

Or configure backend URL: npm start -- --backend http://server-ip:3000

Step 3: Start Streaming!

  • Browse your recorded shows with thumbnails and metadata
  • Select any episode to start playback
  • Progress is automatically saved across all devices
  • Episodes auto-play when one finishes
